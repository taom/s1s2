# S1S2 Planning Artifacts

This folder contains the operating documents that support the main rollout plan.

## Files

- `S1S2-Feature-Entitlement-Matrix.md` - source of truth for free, premium, and coming-later feature status by phase
- `S1S2-Content-Manifest.csv` - tracked species list with phase targets and asset planning status
- `S1S2-Analytics-Event-Taxonomy.md` - event naming rules, required properties, and KPI event coverage
- `S1S2-Release-Gate-Scorecard.md` - gate reviews, thresholds, hold rules, and fallback actions
- `S1S2-Health-Data-Policy.md` - storage, export, deletion, retention, and trust rules for health data
- `S1S2-Support-Playbook.md` - issue classes, severity model, SLAs, escalation, and response rules
- `S1S2-App-Review-Compliance-Pack.md` - App Review positioning, HealthKit copy, privacy label prep, and moderation rules
- `S1S2-Business-Model-Framework.md` - scenario assumptions and the cash-bookings-versus-recognized-revenue model
- `S1S2-Founder-Weekly-Checklist.md` - week-by-week founder operating checklist from kickoff through launch and recurring growth cycles
- `S1S2-Phase-Kanban-Board.md` - Kanban-style task board organized by phase, status bucket, and pull rules

## Usage

- treat the root rollout plan as the master sequence
- treat these documents as working tools for implementation, launch, and live operations
- update the scorecard, entitlement matrix, and business model first whenever a phase scope changes
